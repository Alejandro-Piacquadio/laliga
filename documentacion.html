Tiki-Data — Documentación completa

PHP 8.5 MariaDB 10.4 Bootstrap 5.3 Sin frameworks

Tiki-Data es una aplicación web de estadísticas de LaLiga. Permite a los usuarios registrarse, iniciar sesión y consultar equipos, jugadores, goleadores y la clasificación general de la liga. Está construida con PHP procedural, MariaDB y Bootstrap, sin frameworks.

Arquitectura

laliga-main/ ├── conexion.php ← conexión MySQLi a la BD ├── header2.php ← cabecera + menú compartido ├── footer.php ← footer + JS de Bootstrap ├── index.php ← dashboard (home) │ ├── auth/ │ ├── login.php ← formulario login │ ├── registro.php ← formulario alta usuario │ └── logout.php ← destruye sesión │ ├── equipos.php ← grid de los 20 equipos ├── jugadores.php ← lista de jugadores (filtra por equipo) ├── clasificacion.php ← top 10 equipos por goles │ ├── home/ ← parciales que include index.php │ ├── stats.php ← contadores (equipos, jugadores, goles…) │ ├── top_equipos.php ← podio con los 3 equipos más goleadores │ └── top_goleadores.php← top 3 pichichis │ ├── css/styles.css ← estilos custom ├── img/ ← escudos, logo, iconos ├── js/ ← JS auxiliar └── laliga.sql ← dump de estructura + datos

Flujo de la aplicación

1. El usuario entra en /index.php.
2. Si no hay sesión activa → redirección a /auth/login.php.
3. El usuario inicia sesión o se registra (auth/registro.php).
4. Tras autenticarse, se guarda $_SESSION['usuario'] y vuelve a index.php.
5. El dashboard muestra estadísticas, top de equipos y top de goleadores (parciales en home/).
6. Desde el menú puede navegar a equipos.php, jugadores.php o clasificacion.php.
7. auth/logout.php cierra sesión y vuelve a index.php.

Base de datos

BD: laliga (MariaDB). Tres tablas principales:

TablaCampos claveDescripción
equiposid, nombre, escudo20 equipos de LaLiga.
jugadoresid, nombre, posicion, goles, equipo_id314 jugadores con su equipo y goles.

Núcleo

conexion.php

/conexion.php

Define las credenciales y abre una conexión MySQLi global $con que el resto de archivos reutilizan.

<?php
define('servidor', 'localhost');
define('usuario', 'root');
define('password', '');
define('baseDeDatos', 'laliga');
$con = new mysqli(servidor, usuario, password, baseDeDatos);
?>

Paso a paso:

1. define() crea constantes con la configuración (host, usuario, contraseña, BD).
2. new mysqli(...) abre la conexión y la asigna a $con, que estará disponible en cualquier archivo que haga require_once "conexion.php".

Auth

auth/login.php

/auth/login.php

Formulario de inicio de sesión. Verifica credenciales con password_verify() y guarda el email en $_SESSION.

<?php
require_once "../conexion.php";
if (session_status() === PHP_SESSION_NONE) session_start();
?>
<link rel="stylesheet" href="/css/styles.css">
<link href="https://cdn.jsdelivr.net/.../bootstrap.min.css" rel="stylesheet">

<?php
$error = "";
if ($_SERVER["REQUEST_METHOD"] === "POST") {
    $email = trim($_POST["email"]);
    $password = trim($_POST["password"]);

    if ($email === "" || $password === "") {
        $error = "Todos los campos son obligatorios.";
    } else {
        $sql = "SELECT id, password FROM usuarios WHERE email = ?";
        $consulta = $con->prepare($sql);
        $consulta->bind_param("s", $email);
        $consulta->execute();
        $resultado = $consulta->get_result();

        if ($resultado->num_rows > 0) {
            $dato = $resultado->fetch_assoc();
            if (password_verify($password, $dato['password'])) {
                $_SESSION["usuario"] = $email;
                header("Location: /index.php");
                exit;
            } else {
                $error = "El usuario no existe o el password es incorrecto.";
            }
        } else {
            $error = "El usuario no existe o el password es incorrecto.";
        }
    }
}
?>
<section class="auth-section">
  <div class="auth-card">
    <h1>Iniciar sesión</h1>
    <?php if ($error): ?><p style="color:red;"><?= htmlspecialchars($error) ?></p><?php endif; ?>
    <form method="post">
      <input type="email" name="email">
      <input type="password" name="password">
      <input type="submit" value="Entrar">
    </form>
  </div>
</section>
1. Carga la conexión y arranca sesión.
2. Carga sólo los CSS (no el header), para que en esta pantalla no aparezca el menú.
3. Si la petición es POST: limpia los inputs con trim().
4. Usa una prepared statement (? + bind_param) → previene SQL injection.
5. Compara contraseña con password_verify() contra el hash guardado.
6. Si todo OK → $_SESSION['usuario'] = $email y redirige a /index.php. Si no, muestra error escapado con htmlspecialchars.

auth/registro.php

/auth/registro.php

Alta de nuevos usuarios. Hashea la contraseña con password_hash() antes de insertarla, y rellena el desplegable de equipo favorito desde la BD.

<?php
require_once "../conexion.php";
if (session_status() === PHP_SESSION_NONE) session_start();
?>
<link rel="stylesheet" href="/css/styles.css">
<link href="https://cdn.jsdelivr.net/.../bootstrap.min.css" rel="stylesheet">

<?php
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
  $email    = trim($_POST['email']);
  $password = trim($_POST['password']);
  $plan     = trim($_POST['plan']);
  $equipo   = trim($_POST['equipo_favorito']);
  $hash     = password_hash($password, PASSWORD_DEFAULT);

  $sql = "INSERT INTO usuarios (email, password, plan, equipo_favorito_id)
          VALUES (?, ?, ?, ?)";
  $consulta = $con->prepare($sql);
  $consulta->bind_param('ssss', $email, $hash, $plan, $equipo);
  $consulta->execute();

  header("Location: /index.php");
  exit;
}

$resultado = $con->query("SELECT id, nombre FROM equipos");
?>
<form method="post" action="/auth/registro.php">
  <input type="email" name="email" required>
  <input type="password" name="password" required>
  <select name="plan">
    <option value="gratis">Plan Gratis</option>
    <option value="5">Plan 5EUR</option>
    <option value="10">Plan 10EUR</option>
  </select>
  <select name="equipo_favorito">
    <?php while ($f = $resultado->fetch_assoc()): ?>
      <option value="<?= $f['id'] ?>"><?= htmlspecialchars($f['nombre']) ?></option>
    <?php endwhile; ?>
  </select>
  <input type="submit" value="Registrarse">
</form>
1. Si POST: recoge email, password, plan, equipo_favorito.
2. password_hash($password, PASSWORD_DEFAULT) crea un hash bcrypt seguro.
3. INSERT INTO usuarios con prepared statement (4 strings).
4. header("Location: /index.php"); exit; redirige tras crear el usuario.
5. Si no es POST: hace un SELECT de equipos para llenar el desplegable de equipo favorito.

jugadores.php

/jugadores.php

Lista jugadores. Si recibe ?id=N, filtra por equipo; si no, los muestra todos.

<?php
if (session_status() === PHP_SESSION_NONE) session_start();
if (!isset($_SESSION['usuario'])) {
    header("Location: /auth/login.php");
    exit();
}
require_once "header2.php";
?>
<?php
require_once "conexion.php";
$id = isset($_GET['id']) ? (int)$_GET['id'] : 0;

if ($id === 0) {
    $jugadores = $con->query("SELECT nombre, posicion FROM jugadores");
} else {
    $stmt = $con->prepare("SELECT nombre, posicion FROM jugadores WHERE equipo_id = ?");
    $stmt->bind_param('i', $id);
    $stmt->execute();
    $jugadores = $stmt->get_result();
}

while ($fila = $jugadores->fetch_assoc()) {
    echo "<div class='tarjeta-equipo'>
            <img src='img/unknow.webp'>
            <div>" . htmlspecialchars($fila['nombre']) . "</div>
            <div>" . htmlspecialchars($fila['posicion']) . "</div>
          </div>";
}
?>
1. Comprueba sesión.
2. $id = (int)$_GET['id'] casteo a entero → primera barrera contra SQL injection.
3. Si $id = 0: query directa de todos los jugadores. Si no: prepared statement con bind_param('i', $id).
4. Itera con fetch_assoc() y pinta cada jugador escapando datos con htmlspecialchars.

clasificacion.php

/clasificacion.php

Tabla con el top 10 de equipos según la suma de goles de sus jugadores.

<?php
if (session_status() === PHP_SESSION_NONE) session_start();
if (!isset($_SESSION['usuario'])) {
    header("Location: /auth/login.php");
    exit();
}
require_once "conexion.php";

$consulta = "
    SELECT e.id, e.nombre, COALESCE(SUM(j.goles), 0) AS goles_totales
    FROM equipos e
    LEFT JOIN jugadores j ON j.equipo_id = e.id
    GROUP BY e.id, e.nombre
    ORDER BY goles_totales DESC
    LIMIT 10
";
$resultado = $con->query($consulta);
?>
<?php require_once "header2.php"; ?>
<table class="table">
  <thead><tr><th>Pos</th><th>Equipo</th><th>Goles</th></tr></thead>
  <tbody>
  <?php $p = 1; while ($f = $resultado->fetch_assoc()): ?>
    <tr><td><?= $p++ ?></td><td><?= htmlspecialchars($f['nombre']) ?></td><td><?= (int)$f['goles_totales'] ?></td></tr>
  <?php endwhile; ?>
  </tbody>
</table>
1. Sesión, conexión.
2. Query con LEFT JOIN + SUM(goles) + GROUP BY + ORDER BY DESC LIMIT 10. COALESCE(...,0) garantiza 0 si un equipo no tiene jugadores.
3. Renderiza tabla Bootstrap con un contador $p para la columna posición.

Seguridad y prácticas aplicadas

RiesgoMitigación
SQL InjectionPrepared statements con bind_param en login, registro y filtro por equipo. Casteo (int) en IDs.
XSShtmlspecialchars() en cualquier dato que se imprima desde BD o $_SESSION.
Contraseñas en claropassword_hash(..., PASSWORD_DEFAULT) + password_verify() (bcrypt).
Acceso no autorizadoComprobación de $_SESSION['usuario'] al inicio de cada página protegida, antes de imprimir HTML.
Header already sentLos session_start y comprobaciones de sesión se hacen antes de incluir header2.php (que emite HTML).
Sesiones residualeslogout.php vacía y destruye la sesión.
